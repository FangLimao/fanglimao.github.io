# 国际化与本地化

**国际化与本地化（Internationalization & Localization，简称 I18n/L10n）** ，是对游戏文本进行各种语言的适配，使其能够显示对应的语言。

> Minecraft 是一个面向世界的游戏，它没有任何特定的文化背景。我们并不能忽视 Minecraft 的玩家来自世界各地，使用多种多样的语言。
>
> ——Harbinger

想象一下，如果没有国际化与本地化，我们在玩游戏时就只能用开发团队的母语进行游玩，这无疑给其他地区的玩家带来了不便。

在附加包中，我们需要通过语言文件（`.lang`文件）对游戏文本进行适配。

在本节中，我们将会尝试对上节所述的示例附加包的包名和描述进行本地化。

## 位置

语言文件位于资源包或行为包的`texts`文件夹下：

```
my_addon_bp/
├─ texts/
        ├─zh_CN.lang
        ├─languages.json
        └─en_US.lang
├─ manifest.json
└─ pack_icon.png
my_addon_rp/
├─ texts/
        ├─zh_CN.lang
        ├─languages.json
        └─en_US.lang
├─ manifest.json
└─ pack_icon.png
```

> 行为包中的语言文件只会修改其包名和简介，其余需要在资源包的语言文件中定义

## 格式

语言文件的格式非常简单：

```
## 注释
键名=键值 
```

语言文件中遵循键值对的格式，其中`=`前的内容为键名，后的内容为键值。

两个井号后的内容将会被视为注释，后面的所有文本都都将被忽略，直到下一行。 

键名除了附加包的名称与简介外，其余都可以自由设定。

## 本地化
### 名称与简介
现在，我们尝试对附加包的名称和简介进行本地化。

首先在我们附加包的根目录下创建texts文件夹，然后分别在其下创建`zh_CN.lang`、`en_US.lang`，写入：

```
## zh_CN.lang
pack.description=我的第一个附加包！
pack.name=第一个附加包
```

```
## en_US.lang
pack.description=My first addon！
pack.name=First Addon
```

为了使游戏能够预读取我们的语言文件，我们还需要在texts文件夹下新建一个`languages.json`，然后在里面写入：

```json
[ 
  "en_US",
  "zh_CN"
]
```

现在导入我们的附加包，将语言切换至美式英语，即可看见附加包的名称与描述发生了变化。

### 方块与物品

利用语言文件，也可以为自定义方块、物品添加本地化名称，如：

```
item.<物品ID>=物品本地化名称
tile.<方块ID>.name=方块本地化名称
```

除此之外，我们还可以利用`minecraft:display_name`组件设置物品与方块的本地化字符串，在后文我们会详细讨论。

## 小结

在本节中，我们了解了：

- 本地化的基本知识；
- 如何将附加包名称和描述进行本地化处理。

本地化的应用领域极广，我们可以对附加包中对几乎所有文本内容进行本地化适配。
